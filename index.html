Стек технологий используемых в повседневной работе

Back-end stack

Spring Framework: Создание RESTful сервисов, реализация бизнес логики, примение различных молудей фремворка, интеграция с базами данных и другими сервисами.

Lombok: Упрощение кода моделей с помощью аннотаций.

Maven: Управление зависимостями и сборка проектов.

MVC: Применение шаблона проектирования для разделения логики приложения.

JPA / Hibernate: Организация доступа к данным через ORM.

MySql / PostgreSQL: Работа с реляционными базами данных.

RabbitMQ: Интеграция систем с помощью асинхронных сообщений.

JUnit: Написание модульных тестов для обеспечения качества кода.

Git: Версионный контроль с использованием GitHub и GitLab.

API

REST (Representational State Transfer): Проектирование и разработка RESTful API с акцентом на простоту, масштабируемость и гибкость. Использование HTTP методов (GET, POST, PUT, DELETE) для взаимодействия с серверными ресурсами. Создание интуитивно понятных и документированных API, которые позволяют клиентам легко взаимодействовать с сервисом.

SOAP (Simple Object Access Protocol): Разработка надежных веб-сервисов с использованием SOAP, обеспечивающих высокий уровень безопасности и транзакционности. Применение SOAP для обмена структурированными сообщениями в формате XML между клиентом и сервером. Интеграция сложных бизнес-процессов и систем с помощью стандартизированных протоколов.

WSDL (Web Services Description Language): Описание веб-сервисов с помощью WSDL, предоставляя четкую спецификацию интерфейсов и операций. Автоматизация генерации клиентского и серверного кода на основе WSDL документов. Обеспечение совместимости и взаимодействия между различными веб-сервисами.

Front-end stack

HTML5 / Thymeleaf: Разработка шаблонов страниц, организация контента и его динамическое отображение.

CSS3 / LESS: Применение стилей для визуального оформления элементов интерфейса, использование препроцессоров для упрощения и оптимизации CSS.

Bootstrap 3/4/5: Создание адаптивных и кроссбраузерных интерфейсов с использованием готовых компонентов и сеток.

JavaScript / jQuery: Реализация клиентской логики, обработка событий, взаимодействие с DOM, отправка AJAX-запросов.

WebSocket / STOMP / SockJS: Настройка двустороннего обмена данными между клиентом и сервером в реальном времени, создание интерактивных веб-приложений.

    Я люблю все, что связано с автоматизацией. Поэтому в свободное время автоматизирую процессы внутри разных компаний, вот самые интересные на мой взгляд:

    Организация: Lift universal

    Сайт: lift-universal.uz

    Проблема: Компания специализируется на продаже, установке и техническом обслуживании лифтов и эскалаторов. Ранее вся отчетность и документация по работам с оборудованием велись в одном Excel-файле, который со временем стал чрезмерно объемным, что затрудняло и замедляло работу с ним. Кроме того, стояла задача минимизации бумажных расходов на отчеты, которые необходимо предоставлять заказчикам по завершении каждого вида работ на объектах.

    Решение: Я разработал клиент-серверное приложение на основе Spring Boot с базой данных PostgreSQL и клиентской частью на чистом JavaScript с применением jQuery. Важной задачей для заказчика было отображение каждого объекта в виде точки на карте, что я реализовал с помощью бесплатного плагина Leaflet и карт OpenStreetMap. В приложении предусмотрены различные роли с изолированными функциями: Администратор контролирует все процессы, имеет доступ к дашборду с визуализацией текущих заявок и статусов оборудования, а также может управлять пользователями системы. Модератор отвечает за добавление и редактирование объектов с установленным оборудованием на главной карте. Пользователь используется техническими бригадами для обслуживания объектов и предоставления отчетов заказчику. Менеджер занимается онбордингом клиентов, добавлением маркеров и комментариев к объектам, а также управлением жизненным циклом объектов от статуса к статусу, вплоть до их конвертации в объекты, подлежащие обслуживанию. Каждое оборудование имеет индивидуальный перечень работ, зависящий от модели и необходимого количества обслуживаний в год. Проект был выполнен мной ‘под ключ’, начиная от разработки концепции до загрузки на сервер заказчика, за исключением деплоймента, который осуществлялся девопсом заказчика

    Организация: Сервисный центр по ремонту компьютерного и офисного оборудования

    Проблема: Сеть сервисных центров, раскинувших свои филиалы по различным городам, централизованно осуществляет заказ запасных частей через головной сервисный центр. В период, когда объемы поставок оставались в пределах управляемых, бизнес эффективно функционировал, используя Excel-файлы для оформления заказов и ведения отчетности в центральном офисе. Такой же подход был применен для ведения ‘склада’. Но по мере роста бизнеса использование Excel таблиц становится очень сложной и не удобной задачей.

    Решение: В рамках стремления к цифровой трансформации бизнес-процессов, мной было успешно реализовано масштабируемое клиент-серверное приложение, оптимизирующее управление складскими запасами и заказами. Проект основан на надежном и проверенном технологическом стеке, включающем Spring Framework, PostgreSQL и JavaScript с jQuery, что обеспечивает высокую производительность и удобство использования.

  • Управление Складом: Интуитивно понятный интерфейс для мониторинга истории движения товаров, пополнения запасов и списания материалов.
  • Система Заказов: Гибкая система обработки заказов, поддерживающая как гарантийный ремонт, так и прямые покупки, с возможностью интеграции с поставщиками.
  • Менеджерский Функционал: Эффективные инструменты для подтверждения заказов, их консолидации и последующей отправки, что значительно упрощает логистические процессы.
  • Финансовые Операции: Встроенная система управления балансом пользователей с опцией пополнения и матричной системой комиссий для гибкого ценообразования.

    • Результатом стал уникальный аналог интернет-магазина, специально адаптированный под узкоспециализированные нужды бизнеса, обеспечивающий прозрачность и эффективность всех торговых операций.
